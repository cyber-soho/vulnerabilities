Scan Types Comparison

External Observer View

Connects like a hacker from the outside. Identifies open ports, grabs banners, and infers vulnerabilities based on responses.

Pros: Fast, no credentials needed.
Cons: Higher false positives, misses local config flaws.
Insider View (Trusted)

Logs in using SSH/Windows credentials. Inspects file versions, patch levels, and local registry/config settings directly.

Pros: Highly accurate, detailed, detects missing patches.
Cons: Requires credential management, slower.

Reflection Prompts

Why do Auth scans matter?

Unauthenticated scans only see the outside. Authenticated scans log in to find outdated software versions, bad configs, and missing patches that aren't visible externally.

Ethics of Scanning?

Scans can crash fragile services (DoS). Always have authorization, scan during maintenance windows, and start with "safe" policies in production.
